# Web后台管理系统 - 前端开发文档 (B1组)

## 1. 项目简介
本项目是“多媒体展示软件（仿朋友圈）”的 Web 后台管理端前端部分。
主要用于管理员对注册用户进行管理（查看列表、删除用户），并预留了内容管理功能的接口。
项目严格遵循组长要求，采用 **前后端分离架构**，基于 **本地 Node.js 环境** 进行开发和部署。

## 2. 技术栈
- **核心框架**: Vue 3 (Composition API)
- **构建工具**: Vite (极速冷启动)
- **UI 组件库**: Element Plus (专为后台管理设计)
- **路由管理**: Vue Router 4
- **HTTP 客户端**: Axios

## 3. 功能实现 (B1 职责)
### 3.1 框架搭建
- 实现了经典的后台管理布局：左侧侧边栏导航 (Sidebar) + 顶部通栏 (Header) + 内容区域 (Main)。
- 侧边栏菜单已配置路由跳转，并预留了 B2 组的“内容管理”入口。

### 3.2 用户管理模块
- **页面路径**: `/users`
- **核心功能**:
    - **列表展示**: 使用表格展示用户 ID、用户名、邮箱、注册时间等信息。
    - **删除操作**: 每一行配备“删除”按钮，点击后触发二次确认弹窗，防止误操作。
- **数据交互**: 
    - 当前阶段使用本地 Mock 数据进行演示，确保界面逻辑通畅。
    - 代码中已预留 `axios` 请求代码块，后端接口就绪后取消注释即可对接。

```text
admin-web/
├── public/             # 静态资源

│   ├── router/

```

## 5. 运行指南

### 5.1 环境准备
确保本地已安装 Node.js (建议 v16+)。

### 5.2 安装依赖
首次运行前，请在终端进入 `admin-web` 目录并执行：
```bash
npm install
```

### 5.3 启动开发服务
```bash
npm run dev
```
启动成功后，终端会显示访问地址（通常为 `http://localhost:5173`），在浏览器中打开即可查看效果。

## 6. 后续对接计划
待后端（API开发组）提供服务器地址后，需修改 `src/views/UserList.vue` 文件：
1. 找到 `fetchUsers` 方法，将 Mock 数据替换为 `axios.get('服务器地址/api/users')`。
2. 找到 `handleDelete` 方法，将模拟删除替换为 `axios.delete('服务器地址/api/users/{id}')`。
